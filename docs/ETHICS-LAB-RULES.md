# Ethics & Lab Rules

**Golden Rule:** Only attack or probe systems you own or have explicit permission to test.

## Lab Boundaries
- Use intentionally vulnerable targets (e.g., OWASP Juice Shop) or self-contained samples.
- Keep all credentials and API keys out of source control.
- Avoid scanning or brute forcing anything on external networks.
- Keep logs/evidence sanitized if they include hostnames or usernames you don’t want public.

## Safety
- Snapshot VMs before experiments; revert if needed.
- Document steps and keep evidence (screenshots, logs) in each week’s folder.
- Prefer least-privilege and isolated networks.
